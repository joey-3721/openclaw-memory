## 2026-03-20 Media Hub discover/filtering + deletion fixes
- Fixed library delete modal root cause: `pendingDeleteId` had a JS scope bug in `library.html`; delete button handler referenced it outside the `DOMContentLoaded` local scope, causing no-op/ReferenceError. Resolved by promoting `pendingDeleteId` to script scope. Commit: `55926e6`.
- Reverted library/watchlist infinite scroll back to bottom pagination after user feedback; default page size is now 20 and pagination stays at the bottom. Discover page pagination also moved to the bottom. Commit: `d4dd3e5`.
- Refined watchlist watch-toggle UX: second click now reverts watched -> wish and toast says `已撤销 ↩️` instead of incorrectly saying watched again.
- Discover page filter model was upgraded from hardcoded static filters to TMDB-oriented filters with genre support. Commit: `f8d61f0` initially added TMDB genre chips + free year input.
- User rejected free-text year input as awkward; year filter changed to rolling year chips based on current year (2026..2020) plus decade ranges (`2010s`, `2000s`, `1990s`). Backend maps decade chips to TMDB date-range params. Commit: `f5bf91c`.
- User wants discover genres ordered by personal taste profile instead of arbitrary order, with less-used genres hidden behind an expand button. I analyzed Douban watch history genre frequency from local SQLite and reordered discover genre chips accordingly.
  - Movie preference order from watch history: 冒险 > 动作 > 喜剧 > 奇幻 > 科幻 > 剧情 > 动画 > 惊悚 > 悬疑 > 爱情 > 犯罪 > 古装 > 恐怖 ...
  - TV preference order from watch history: 剧情 > 悬疑 > 奇幻 > 动作 > 惊悚 > 古装 > 恐怖 > 爱情 > 犯罪 ...
  - Default visible genre chips now show the first 7 preferred genres; the rest are hidden behind `更多 ▾` / `收起 ▴`. `动画/动漫` was intentionally moved later per user request. Commit: `ed6d603`.
- Important current discover-page behavior: genre ordering is now taste-driven and collapsible, year chips are rolling instead of hardcoded, and TMDB remains the primary discovery source.
