## Media Hub 推荐系统修复

### 修复：INSERT ON DUPLICATE KEY UPDATE（不覆盖用户字段）

**问题**：`REPLACE INTO` = DELETE + INSERT，会抹掉用户后来修改的 status/my_rating/comment/watch_count。
- 「窥探」(tmdb_tv_117378)：用户标记为看过（collect），被 cron 覆盖回 recommended
- 「你想活出怎样的人生」(tmdb:movie:508883)：用户设为推荐，MySQL INSERT 时字段不完整导致 status 丢失

**根因**：Python 字符串里写了 `-- 注意：不更新...` SQL 注释，被 pymysql 当成 SQL 注释截断，ON DUPLICATE KEY UPDATE 只有前半截生效，等效于覆盖。

**修复内容** (`add_one_recommendation.py`)：
1. `insert_recommendation()` 函数：彻底重写为单行拼接 SQL 字符串，去掉所有 `--` 注释，ON DUPLICATE KEY UPDATE 只包含推荐相关字段（title/kind/year/.../recommendation_note/recommend_rank/douban_rating...），故意省略 status/my_rating/comment/watch_count
2. 去重检查：新增 `status='collect'` 拦截逻辑，已看过条目绝对不推荐；`status='recommended'` 跳过；只有全新/wish 才允许插入
3. subject_id 格式统一：`tmdb:tv:` 和 `tmdb:movie:` 前缀（旧数据是 `tmdb_tv_xxx` 和 `tmdb_xxx`，MySQL 迁移遗留）
